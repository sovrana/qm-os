## Draft Option A: The Problem Frame

Every knowledge worker has the same problem. You spend more time coordinating - filing, retrieving, prioritising, remembering context - than you spend on the work that actually requires your judgement. AI assistants help with individual tasks but forget everything between conversations.

Quartermaster fixes this. It's an agentic operating system built on Claude Code that turns a markdown vault into a persistent, self-improving assistant. It remembers what you decided last week, knows how your stakeholders prefer communication, catches your blind spots before you ship, and gets measurably smarter every week through a built-in feedback loop.

Built over 8 weeks of daily production use. Not a proof of concept.

## Draft Option B: The Builder Frame

I built an agentic operating system for my work. Here's how.

Claude Code is powerful but stateless. Every conversation starts from zero. I needed something that compounds - where every correction becomes a permanent rule, every session builds institutional memory, and workflows that took 30 minutes shrink to 5.

Quartermaster is the result: 7 published skills, a three-hook automation layer, a six-file persistent memory system, local semantic search over a markdown vault, and a self-improvement loop that promotes recurring patterns into system rules.

This site documents the architecture and methodology. The template repo gives you the working system.

After 8 weeks of daily use, my CLAUDE.md file contains dozens of rules I never wrote. They were discovered through use - corrections that became patterns, patterns that became permanent instructions.
