# About

## Who Built This

Marc Lien. I run AI transformation programmes for professional services businesses inside a global private equity firm.

My day job involves advising portfolio company boards on how to move from incremental automation to genuine AI-native operations - new operating models, new economics, new ways of working. Quartermaster is the system I built to manage that work.

Before PE, I spent 15 years in operational architecture and business model transformation. The common thread: turning messy strategic problems into systematic, repeatable solutions. QM is the latest version of that instinct, applied to my own workflow.

## Why This Exists

I was managing five active workstreams across multiple portfolio companies, each with different stakeholders, different communication styles, different political dynamics. Every morning I'd open Claude Code and spend the first ten minutes re-explaining context it had yesterday.

I needed an AI assistant that compounds. One where every session makes the next one better. Where a rejected framing becomes a permanent constraint. Where a cross-theme insight gets captured, not forgotten. Where the system literally rewrites its own instructions based on how I work.

Nothing like this existed. So I built it.

## The Timeline

8 weeks of daily production use across multiple active workstreams. Not a weekend project.

**Week 1-2:** Core vault structure, task management, first skills (`/morning`, `/inbox`).
**Week 3-4:** Memory system, semantic search, hooks automation.
**Week 5-6:** Advanced skills (`/challenge`, `/weekly`, `/draft`), self-improvement loop.
**Week 7-8:** Calibration system, rules architecture, voice matching, privacy-first publishing.

The CLAUDE.md file now contains dozens of rules that were discovered through use, not designed upfront. That's the point.

## The Thesis

Knowledge workers spend most of their time on coordination: filing, retrieving, prioritising, remembering context, adapting communication style per audience. AI agents can handle all of this, if you architect the system.

The same principle applies at enterprise scale. The professional services firms I work with have thousands of people doing coordination work that follows patterns. The question isn't whether AI can do this work - it's whether you build the architecture that lets it compound.

Quartermaster is that architecture at the individual level. Open source. MIT licensed. Built to be forked and adapted.
